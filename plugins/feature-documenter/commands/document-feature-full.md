# Document Feature (Full)

You are executing the `/document-feature-full` command.

@../SKILL.md

Create feature documentation using the **comprehensive template**.

## When to Use

- Complex features with multiple components
- Features spanning multiple systems
- Documentation for team reference
- Features with architectural implications
- When risk assessment is needed

## Arguments

- `$ARGUMENTS` — Feature name or file path to document

## Template Reference

Use `references/template-comprehensive.md` as the base structure.

## Sections

1. **Problem Statement** — Detailed problem analysis
2. **Solution Overview** — Complete solution architecture
3. **Core Components** — All components with responsibilities
4. **Dependencies** — External and internal dependencies
5. **Data Flow** — How data moves through the feature
6. **API Contracts** — Inputs, outputs, interfaces
7. **Error Handling** — Failure modes and recovery
8. **Testing Strategy** — How to test the feature
9. **Risk Assessment** — Known risks and mitigations
10. **Future Considerations** — Planned enhancements

## Output

Comprehensive feature documentation suitable for complex features and team handoffs, saved to the project's docs folder.
